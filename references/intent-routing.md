# 意图路由与风险规则

## 路由步骤

1. 判断动词：记录、整理、复盘、分析、更新、创建、删除。
2. 判断对象：身体、情绪、时间、项目、财务、学习、知识、档案。
3. 判断输出：新增记录、修改档案、生成计划、整理文件、生成复盘。
4. 判断风险：追加、改写、移动、合并、删除、敏感推断。
5. 限定读取范围。
6. 执行或请求确认。

## 意图表

| 意图 | 典型请求 | 默认范围 | 策略 |
| --- | --- | --- | --- |
| `record.body` | 记录训练、身体指标、饮食、睡眠 | `30-数据记录/身体/` | 追加直接写；更新健康档案需确认 |
| `record.mood` | 记录心情、压力、精力、触发因素 | `30-数据记录/情绪/` | 追加直接写；心理结论只标观察 |
| `record.time_project` | 记录时间投入、项目进展 | `30-数据记录/时间与项目/`、`20-学习与项目/` | 追加直接写；改项目状态需预览 |
| `record.finance` | 记录收支、资产负债、预算目标 | `30-数据记录/财务/` | 原始记录直接写；财务状态需确认 |
| `profile.update` | 更新当前目标、能力、价值观、状态 | `10-个人档案/` | 必须展示差异 |
| `learning.loop` | 建学习计划、项目、练习、测验 | `20-学习与项目/`、`40-知识库/` | 新建可直接；改既有计划需预览 |
| `knowledge.organize` | 整理收件箱、补标签、合并重复 | `40-知识库/` | 元数据可直接；移动合并删除需确认 |
| `review.pattern` | 周复盘、月复盘、找模式 | 按时间窗口读取相关记录 | 草稿可直接；档案结论需确认 |
| `state.diagnose` | 分析效率低、压力、拖延、恢复 | 近期身体、情绪、时间项目、目标 | 只输出可能关联 |
| `decision.support` | 帮我决定、整理利弊、风险 | 目标、项目、证据、情绪压力 | 草稿可直接；落档需确认 |
| `review.to_action` | 把复盘变成下周行动 | 复盘、目标、行动实验 | 新建行动实验可直接；改目标需确认 |

## 风险分级

低风险，可直接执行：

- 新增记录。
- 补 Properties。
- 补标签。
- 补双链。
- 生成草稿和派生视图。

中风险，先预览：

- 移动文件。
- 重命名文件。
- 批量调整分类。
- 更新项目状态。

高风险，单独确认：

- 删除文件。
- 合并文件。
- 覆盖既有内容。
- 修改大量历史记录。
- 涉及健康、财务等敏感档案的推断性改写。

## 歧义处理

- 优先写原始记录，不急着改规范档案。
- 可以拆成多条记录时拆分。
- 需要改档案时生成“建议更新”并展示差异。
- 不确定时只问一个短问题。

## 高风险领域边界

健康、心理、财务相关输出必须避免专业诊断或投资建议。表达方式应为：

- “可能关联”
- “观察到”
- “记录支持”
- “需要继续观察”
- “可验证的小实验”

避免：

- “确定是”
- “医学上说明”
- “你应该投资”
- “保证有效”
