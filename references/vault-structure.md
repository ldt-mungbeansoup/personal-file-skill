# Vault 结构规范

## 顶层目录

```text
00-首页与导航
10-个人档案
20-学习与项目
30-数据记录
40-知识库
90-系统
```

## 00-首页与导航

用途：用户每天打开 Vault 时的个人驾驶舱。

建议文件：

```text
00-首页与导航/README.md
00-首页与导航/首页.md
00-首页与导航/快速记录入口.md
00-首页与导航/当前状态总览.md
00-首页与导航/近期复盘索引.md
```

首页包含：

- 当前状态。
- 快速记录入口。
- 正在推进。
- 近期复盘。
- 系统说明。

## 10-个人档案

用途：动态自我模型和事实档案。

建议结构：

```text
10-个人档案/README.md
10-个人档案/基本信息.md
10-个人档案/当前目标.md
10-个人档案/能力地图.md
10-个人档案/兴趣偏好与价值观.md
10-个人档案/健康与身体状态.md
10-个人档案/财务状态.md
10-个人档案/外部反馈.md
10-个人档案/决策记录/
```

规则：

- 规范档案记录当前状态。
- 改写前必须展示差异。
- 重要结论链接到证据来源。

## 20-学习与项目

用途：学习目标、实践项目、问题清单、练习测验、阶段复盘和行动实验。

建议结构：

```text
20-学习与项目/README.md
20-学习与项目/学习目标/
20-学习与项目/项目/
20-学习与项目/问题清单/
20-学习与项目/练习测验/
20-学习与项目/阶段复盘/
20-学习与项目/行动实验/
```

## 30-数据记录

用途：原始数据和周期复盘。

```text
30-数据记录/README.md
30-数据记录/身体/身体指标/
30-数据记录/身体/训练/
30-数据记录/身体/饮食/
30-数据记录/身体/睡眠与恢复/
30-数据记录/情绪/心情记录/
30-数据记录/情绪/精力与压力/
30-数据记录/情绪/触发因素与应对/
30-数据记录/时间与项目/时间投入/
30-数据记录/时间与项目/项目进展/
30-数据记录/时间与项目/阶段复盘/
30-数据记录/财务/收入与支出/
30-数据记录/财务/资产与负债/
30-数据记录/财务/预算与财务目标/
```

规则：

- 原始记录追加优先。
- 周/月/阶段复盘必须列出来源。
- 不把观察写成绝对因果。

## 40-知识库

用途：概念、问题、资料、方法和收件箱整理。

建议结构：

```text
40-知识库/README.md
40-知识库/收件箱/
40-知识库/概念/
40-知识库/问题/
40-知识库/资料/
40-知识库/方法/
40-知识库/索引/
```

## 90-系统

用途：模板、字段说明、助手使用说明、变更日志和回滚说明。

建议结构：

```text
90-系统/README.md
90-系统/模板/
90-系统/字段说明.md
90-系统/助手使用说明.md
90-系统/命名规则.md
90-系统/变更日志/
```

系统目录不是垃圾桶。只放维护规则和系统说明。
